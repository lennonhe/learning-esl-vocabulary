## Key Differences from Comprehensive Entry

| Aspect | Comprehensive Entry | Context Explanation |
|--------|---------------------|---------------------|
| Scope | All meanings | Only the meaning in context |
| Header | `# {word}` | `# {word} (in context)` |
| Original sentence | Not included | Included at top |
| Meanings section | `## Meanings` with numbered meanings | `### Meaning in this context` (single) |
| Example sentences | Generated for each meaning | User's sentence first, then similar examples |
| Common patterns | All patterns for all meanings | Only patterns for this specific meaning |
| File naming | `{word}.md` | `{word}-in-context.md` |

## File Naming Rules

- Save as `{word}-in-context.md` in today's date folder
- If file already exists, add sequence postfix: `{word}-in-context-2.md`, `{word}-in-context-3.md`, etc.
- Use lowercase for the word
- For multi-word targets, use hyphens: `work-out-in-context.md`

## Formatting Guidelines

- Use lowercase for the title (word) - do not capitalize
- Include "(in context)" in the title to distinguish from comprehensive entries
- Always include the original sentence at the top
- Mark the original sentence with "*(Original)*" in the examples list
- Focus only on the meaning relevant to the given context
- Keep memory tips specific to this usage, not general tips for all meanings
